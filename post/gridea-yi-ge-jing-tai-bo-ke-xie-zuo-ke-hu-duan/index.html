Gridea—一个静态博客写作客户端

2019-04-17

简介

Gridea（旧称 Hve Notes）是一个静态博客写作客户端 (A static blog writing client) ，你可以用它来记录你的生活、心情、知识、笔记、创意等等。配合github账号你可以搭建一个静态页面的博客，而且界面还非常美观哦，写作推送也是非常的方便。

演示地址：https://gridea.dev/ 项目地址：https://github.com/getgridea/gridea

特性

1、你可以使用最酷的 Markdown 语法，进行快速创作 2、你可以给文章配上精美的封面图和在文章任意位置插入图片 3、你可以对文章进行标签分组 4、你可以自定义菜单，甚至可以创建外部链接菜单 5、你可以在Windows或MacOS设备上使用此客户端 6、你可以使用Github pages或 Coding Pages向世界展示，未来将支持更多平台 7、你可以进行简单的配置，接入 Gitalk 或 DisqusJS 评论系统 8、你可以使用中文简体或英语 9、你可以任意使用应用内默认主题或任意第三方主题，强大的主题自定义能力 10、你可以自定义源文件夹，利用 OneDrive、百度网盘、iCloud、Dropbox 等进行多设备同步

依赖

需要提前安装 git ，因为这个客户端是用git来推送文章到Github pages或 Coding Pages的

使用

1、下载安装客户端 支持Windows或MacOS，自行选择一路安装就可以了，安装后打开客户端如下图

2、在github新建个repository

3、配置域名

3.1配置之前我们随便在项目里建一个文件，因为空的项目无法选择Github pages 新建个readme吧 直接commit new file即可

3.2 设置为Github pages

选择master branch,然后就可以看到访问的域名了，当然下面也可以自定义域名，我们这里自定义为自己的域名6v.nu，然后还需要把6v.nu设置CNAME解析到swordstudiox.github.io

4、配置客户端 进入应用的配置界面，进行应用的初始化配置。下面以 Github Page 为例。

4.1 域名：Github Pages 对应的域名（例如：https://swordstudiox.github.io/blog 或 http://6v.nu（这里为自定义域名））

4.2 仓库：静态文件存放的仓库（例如：swordstudiox.github.io 或 blog(这里仓库名)）

4.3 分支：Github Pages 对应的分支（例如：master）

4.4 用户名：仓库所属用户的用户名（例如：swordstudiox，同时也是 Git Repo 中配置 git config user.name "用户名"）

4.5 邮箱：Git 推送时的邮箱，等同于在 Git Repo 中配置 git config user.email "邮箱"

4.6 CNAME: 若网站有自定义域名，则可通过此设置（例如：6v.nu)

4.7 Token: Git 推送时需要的 Token，用来向 Github 提交构建后的文件 Github personal access tokens,勾选上 repo 的相关权限即可

4.8 最后点击客户端左下角的同步按钮，静待一会就OK了 搭建成功地址 https://6v.nu/
